Seccion de preguntas a responder

  • ¿Cuál es la diferencia entre Internet y la World Wide Web?

    • Respuesta: El Internet es la infraestructura detras de cables, routers y servidores que permite la conectividad global, mientras que la World Wide Web es un servicio específico en Internet que proporciona acceso a páginas web y otros recursos multimedia a través de navegadores web.

  • ¿Cuáles son las partes de un URL?

    • Respuesta: Protocolo (http://), nombre de dominio(.com), subdominio (www, este es opcional), ruta(Es la ubicación específica del recurso en el servidor, es decir el URL) y fragmento(Es una referencia a una ubicación específica dentro del recurso, como un marcador en una página web, # seguido del fragmento.).

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    • Respuesta: Estos métodos proporcionan una forma estándarizada de interactuar con recursos en servidores web, lo que permite a los clientes (navegadores, aplicaciones, etc.) realizar una amplia variedad de acciones en la web de manera consistente y segura.

    GET: Solicita datos.

    HEAD: Solicita solo encabezados de respuesta.

    POST: Envía datos al servidor.

    PUT: Reemplaza completamente un recurso.

    PATCH: Aplica modificaciones parciales.

    DELETE: Elimina un recurso.

  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    • Respuesta: Al enviar datos sensibles como un usuario y contraseña, es mejor utilizar el método POST debido a su capacidad para enviar datos de forma segura y evitar exponer información confidencial en la URL.

  • ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    • Respuesta: Cuando abres una página web en tu navegador escribiendo la dirección en la barra de búsqueda y presionas "Enter", el navegador utiliza el método GET para solicitar la página al servidor. GET es como una solicitud de información: le pide al servidor que te envíe la página web y todos sus elementos, como imágenes y texto, para que puedas verlos en tu navegador.

  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    • Respuesta: Significa que el servidor ha recibido y procesado correctamente la solicitud del cliente, y está devolviendo el contenido solicitado, por lo tanto no ocurre ningun error.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    • Respuecsta: Sí, es responsabilidad del desarrollador corregir un sitio web si un usuario reporta un error 404. El error 404 indica que el servidor no pudo encontrar el recurso solicitado, lo que generalmente significa que la página web o el archivo no existe en la ubicación especificada.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    • Respuesta: Sí, es responsabilidad del desarrollador corregir un sitio web si un usuario reporta un error 500. El error 500, también conocido como "Error interno del servidor", indica que hubo un problema en el servidor mientras procesaba la solicitud del usuario. Este tipo de error no se debe a un problema con la solicitud del cliente, sino a un problema interno en el servidor.

  • ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    • Respuesta: Cuando un atributo o elemento HTML5 está depreciado o desaprobado, significa que se considera obsoleto y se desaconseja su uso en el desarrollo de nuevas páginas web. Algunos elementos de HTML 4 que están desaprobados en HTML5 incluyen font, center, frame, frameset, noframes, applet, y acronym. Su uso puede causar problemas de compatibilidad y no es recomendado para garantizar el cumplimiento de los estándares web modernos.

  • ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    • Respuesta: HTML5 es una versión mejorada de HTML4. Tiene nuevas herramientas para hacer páginas web más bonitas, como añadir música o videos sin necesidad de programas externos, dibujar gráficos, y hacer formularios más fáciles de usar. También ayuda a que las páginas sean más fáciles de encontrar en Internet y más accesibles para personas con discapacidades.

  • ¿Qué componentes de estructura y estilo tiene una tabla?

    • Respuesta:

    Componentes de estructura: table( Este elemento define la tabla en sí misma), tr(Representa una fila dentro de la tabla.), td(Define una celda de datos en una fila.) y th(: Define una celda de encabezado en una fila. )

    Componentes de estilo: CSS(Se utiliza para aplicar estilos visuales a la tabla y sus elementos, como colores de fondo, bordes, márgenes, etc.) y atributos de estilp HTML(ej. style="background-color: #ffffff;).

    Otros elementos basicos que puede contener una tabla son: caption, colgroup, col, thead, tbody, tfoot.

  • ¿Cuáles son los principales controles de una forma HTML5?

    • Respuesta: Los principales controles de formulario en HTML5 incluyen input, textarea, select y button, estos elementos permiten a los usuarios ingresar y seleccionar datos en un formulario web; al igual que HTML5 introduce nuevos tipos de entrada y atributos de validación para mejorar la experiencia del usuario y la funcionalidad de los formularios.

    Sobre el ciclo de vida y desarrollo de los sistemas de información:

  • ¿Cuál es el ciclo de vida de los sistemas de información?

    • Respuesta: El ciclo de vida de los sistemas de información abarca las etapas de identificación de necesidades y oportunidades, análisis, diseño, desarrollo e implementación, operación y mantenimiento, y finalmente, retiro. Comienza con la identificación de las necesidades del negocio, seguida del análisis detallado de requisitos, el diseño de la arquitectura del sistema, la construcción del sistema, su operación y mantenimiento continuo, y concluye con el retiro del sistema cuando ya no es útil o efectivo.

  • ¿Cuál es el ciclo de desarrollo de sistemas de información?

    • Respuesta: El ciclo de desarrollo de sistemas de información, también conocido como ciclo de vida del desarrollo de software, consta de las siguientes etapas: requerimientos, diseño, desarrollo, pruebas, implementación y mantenimiento. Comienza con la recopilación de requisitos, seguida del diseño del sistema, el desarrollo del software, pruebas exhaustivas, implementación en el entorno operativo del cliente y mantenimiento continuo para corregir errores y realizar mejoras. Este ciclo puede variar según la metodología utilizada, como el modelo en cascada o metodologías ágiles como Scrum y DevOps.

    Preguntas Lab2 (CSS)

  • Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

    • Respuesta: Su uso debe ser moderado y reservado para casos específicos; de lo contrario, el código puede tener fallas y ser difícil de mantener, esto se debe a que puede resultar en reglas difíciles de modificar y problemas de especificidad en cascada.

  • Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

    • Respuesta: Elegir una imagen de fondo en una página web es importante por varias razones; la imagen no debe distraer o dificultar la lectura del contenido, debe verse bien en diferentes dispositivos y cargarse rápidamente; también debe ser accesible para todos los usuarios y complementar el diseño general de la página.

  • Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

    • Respuesta: Al momento de elegir las unidades de una propiedad de estilo se debe pensar bien en que es lo que se necesita o requiere, dependiendo de lo que se este disenando y que quieras que se vea en diferentes dispositivos; ya que los % son buenos para hacer diseños que se ajusten al tamaño de la ventana o el contenedor, los px son para tener control exacto sobre el tamaño y la posición de los elementos; y los pt son más para cosas impresas y medidas físicas concretas.

  • ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

    • Respuesta: Usar una versión minimizada del CSS hace que el sitio cargue más rápido porque el archivo es más pequeño; esto significa menos tiempo de espera para los usuarios y menos estrés para el servidor; además, el navegador puede procesarlo más rápido, lo que hace que la experiencia de navegación sea más fluida.
