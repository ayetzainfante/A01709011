Diferencias y Semejanzas

Diferencias
  • Java es un lenguaje de programación de propósito general, orientado a objetos y de tipado estático, mientras que JavaScript es un lenguaje de programación interpretado, orientado a objetos y de tipado dinámico.
  • Java se utiliza principalmente para aplicaciones de servidor, desarrollo de aplicaciones de escritorio y Android, mientras que JavaScript se usa principalmente para desarrollo web, tanto en el lado del cliente como del servidor.
  • Java requiere compilación antes de la ejecución, mientras que JavaScript se interpreta en tiempo de ejecución por el navegador o el motor JavaScript del servidor.

Semejanzas
  • Ambos lenguajes están orientados a objetos.
  • Comparten algunas estructuras de control de flujo como if, for, while, etc.
  • Tienen sintaxis similar en algunas áreas debido a su influencia compartida de C.

Metodos del objeto Date

  1. getDate(): Obtiene el día del mes (del 1 al 31).
  2. getMonth(): Obtiene el mes (del 0 al 11).
  3. getFullYear(): Obtiene el año con cuatro dígitos.
  4. getHours(): Obtiene la hora (0-23).
  5. getMinutes(): Obtiene los minutos (0-59).

Metodos de los arreglos

  1. push(): Añade uno o más elementos al final del arreglo y devuelve la nueva longitud del arreglo.
  2. pop(): Elimina el último elemento de un arreglo y devuelve ese elemento.
  3. splice(): Cambia el contenido de un arreglo eliminando o reemplazando elementos existentes y/o agregando nuevos elementos.
  4. forEach(): Ejecuta una función proporcionada una vez por cada elemento del arreglo.
  5. filter(): Crea un nuevo arreglo con todos los elementos que pasan el test implementado por la función proporcionada.

¿Cómo se declara una variable con alcance local dentro de una función?

Al declarar una variable con alcance local dentro de una función, simplemente defines la variable dentro del cuerpo de la función; es decir, que la variable solo será accesible y existirá dentro de esa función en particular. Fuera de esa función, la variable no estará disponible para su uso.

Implicaciones de utilizar variables globales dentro de funciones

  • Contaminación del espacio global: Las variables globales pueden ser accedidas y modificadas desde cualquier parte del código, lo que puede llevar a colisiones de nombres y comportamiento inesperado.
  • Dificultad para depurar: Cuando se utilizan muchas variables globales, puede ser difícil rastrear su origen y cómo cambian a lo largo del programa, lo que dificulta la depuración.
  • Mala práctica de programación: Usar variables globales puede indicar un diseño deficiente, ya que puede hacer que el código sea menos modular y más difícil de mantener.
  • Posible pérdida de encapsulamiento: Al tener acceso global, se pierde el encapsulamiento y la modularidad, lo que puede dificultar la comprensión y la reutilización del código.
